# OPC Obsidian OS

> 一个 **AI 一人公司（One-Person Company）的内容运营操作系统**：用 Obsidian 当数据库，用一份 `AGENTS.md` 让任何 AI 工具当你的运营团队，用一个本地看板看进度。**作者真在用，不是 demo。**

一个人做 IP / 产品，最难的不是写代码，是**每天要记住：哪条选题写到哪了、发到哪些平台、数据怎么样、下一步该干嘛**。这个项目把整套流程压成 **一张表 + 一套规约 + 一个看板**。

## 三个核心文件

| 文件 | 作用 |
|---|---|
| **`_首页.md`** | 你唯一要看的**表盘**。选题流水线、产品、账号全在这，状态由 AI 维护，你只管拍板。 |
| **`AGENTS.md`** | AI 执行规约。换任何 AI 工具（Claude Code / Cursor / ChatGPT），让它先读这个，它就按你的规矩干活。 |
| **`看板/`** | 双击启动的本地 **Kanban**，直接读写 `_首页.md`：看进度、点按钮推进、同步文件、增删改选题。 |

## 核心理念

| 理念 | 说明 |
|---|---|
| **单一数据源** | 账号 / 产品 / 选题只在首页存一份，其他文件 wiki 链接引用，改一处全局生效 |
| **AI 维护状态，用户只做三件事** | 给 idea / 审文案 / 配音 —— 只做不可外包的判断，其余 AI 做 |
| **5 格流水线** | 选题 → 写稿 → 审核 → 配音 → 发布，每格一个文件夹，AI 自动往前推 |
| **数据回流闭环** | 发布后数据回填复盘表，喂回下一轮选题 |
| **登录态隔离** | 抓平台数据只在受控浏览器里做，绝不碰系统主浏览器的凭据 |

## 适合谁

- 一个人做内容 / 个人 IP / 独立产品的开发者或创作者
- 想验证「一个人 + AI 能不能跑起 内容→产品→增长→变现」的 OPC 实践者
- 已经在用 Obsidian、想要一套现成运营框架的人

## 快速开始

1. **拖进 Obsidian**：把整个文件夹丢进你的 Vault，用 Obsidian 打开。
2. **填首页**：打开 `_首页.md`，把定位 / 账号 / 产品换成你自己的。
3. **让 AI 读规矩**：把 `AGENTS.md` 喂给你的 AI 助手（Claude Code / Cursor 等），它就按这套流程干活。
4. **（可选）开看板**：装好 [Node.js](https://nodejs.org/)（v18+），双击 `看板/启动看板.command`，浏览器里看 Kanban、点按钮推进；稿件和发布包会跟着归位。

> 不装 Obsidian 插件、不联网、看板只绑 127.0.0.1。换 AI 工具零成本（都读同一份 AGENTS.md）。

## 目录结构

```
├─ _首页.md            # 你唯一看的表盘
├─ AGENTS.md          # AI 执行规约（换任何 AI 工具都先读它）
├─ 看板/              # 本地 Kanban 看板
│  ├─ board.html      #   前端页面
│  ├─ server.js       #   读写 _首页.md 的本地服务（零依赖）
│  └─ 启动看板.command #   双击启动（macOS）
├─ 01-输入池/         # idea、热点文章
├─ 02-文案生产/       # 待审核 / 已通过 / 需要修改
├─ 03-配音发布包/     # 待配音 / 已发布
├─ 04-素材资产/       # 出镜 IP 资产（角色设定、动作库）
├─ 05-复盘商业化/     # 发布复盘、商业化记录
└─ 99-归档/           # 废弃
```

## 铁律
- 一律中文，说人话，从用户角度出发。
- 用户只负责三件事：给 idea、审文案、配音。其余 AI 做。
- 别让用户维护复杂表格；所有状态写进 `_首页.md` 的表里，AI 负责更新。
- 用户给文章 / 链接 / 热点时，先判断意图：找资源 / 讨论观点 / 拆对标 / 做热点。别默认每篇都写成文案。
- 蹭热点必须能绑回主线，绑不上就不蹭。不做新闻搬运号。
- 别让自研产品抢走主线。
- **单一数据源**：账号 / 产品 / 选题信息**只在 `_首页.md` 存一份**；其他文件用 `[[_首页#账号]]` 这类 wiki 链接引用，**不重复写账号名**。改首页 = 全局生效。
- **登录态只用受控浏览器，绝不碰系统主浏览器**：小红书 / X / Reddit 等登录态平台，统一在受控浏览器（如 ego-browser）里复用登录态。不要让脚本去读系统 Chrome —— 会动到用户主浏览器的全部凭据（高风险）。
- **抓自己数据用同一空间**：抓各平台发布数据，统一用同一个 task space，不要每个平台开一个空间。

## 分发规则
- 产出**文章** → 发 小红书 + B站 + 公众号 + 知乎。
- 产出**视频** → 发 小红书 + B站；公众号、知乎 发这条视频的文案（改成文章）。
- 公众号和知乎同调性：有文章发文章，只有视频就发视频文案。
- 一条内容的内核可同时出视频和文章。

## 工作流（一条内容从 idea 到发布的闭环）

一条内容 = 表盘里的一行。它从左到右走过 5 个格，每走一格 = 一个文件夹 + 一个动作。AI 每做完一步，就把那行的状态往前推一格。

| 阶段 | 谁触发 | AI 做什么 | 文件落到 | 表盘 |
|---|---|---|---|---|
| ① 选题 | 你给 idea / 丢链接 | 判断能不能做、文章还是视频 | `01-输入池/` | 选题 💡/🔵 |
| ② 写稿 | 你说「做第 X 条」 | 按 `02-文案生产/文案模板.md` 写稿，文件名＝选题名 | `02-文案生产/待审核/{选题名}.md` | 写稿 🟢 |
| ③ 审核 | AI 写完喊你 | 你过目：通过 / 要改 | 通过→`已通过/`；改→`需要修改/` | 审核 🟢 |
| ④ 配音 | 你审通过 | 按 `03-配音发布包/发布包模板.md` 整理（配音稿 + 各平台标题/简介/标签） | `03-配音发布包/待配音/` | 配音 🟢 |
| ⑤ 发布 | 你配完音、发完各平台 | 发布包移到已发布，在 `05-复盘商业化/发布复盘.md` 记一笔 | `03/已发布/` + `05/` | 发布 ✅ |

发完后数据回流到复盘表，喂回下一轮选题——这就是循环。

### 表盘同步（AI 每次操作后必做）
- 把那行的状态格往前推：⬜ → 🟢 → ✅。
- ② 写完稿：把表盘该行**选题名改成可点击链接**，指向 `02-文案生产/待审核/{选题名}.md`，用户在首页一点就跳过去看。
- ③ 审核通过：链接改指向 `已通过/{选题名}.md`。
- ⑤ 发布完：去 `05/发布复盘.md` 记一笔数据。

> 约定：每条选题的稿件文件名＝选题名（如 `让AI当我的运营编辑.md`），路径可预测，链接才能对上。

**idea 的两种进法**
- 当场做：你说「写成一期：XXX」→ 跳过输入池，直接进 ②。
- 先存着：你丢个粗糙想法 → AI 判断后加进表盘选题池（①），等你拍板。

**外部文章 / 链接**
你发链接 → AI 先判断意图（找资源 / 讨论观点 / 拆对标 / 做热点）。只有「做热点、且绑得上主线」才进 ①，其余按你实际需求处理，不默认写成文案。

## 小红书数据抓取（自己的号，发布后统计赞/藏/评）

### 核心原则
- **不存死链接**：帖子链接带 token，token 会变会过期。每次从该账号**主页拿笔记的带 token 链接**（主页当场给，有效），再打开。
- **纯读 DOM，不碰 API**：受控浏览器打开笔记页，执行 JS 读页面上已渲染的赞/藏/评。零接口调用，不被风控。
- **低频**：每条笔记只在发布后 **24h / 48h / 7天** 抓 3 次。
- **别关用户的浏览器**：换页用复用当前 tab 的方式，**绝不关 tab**、别开一堆 tab 又关。

### 数据放哪
- 账号主页 URL → `_首页.md` 的「账号」表。
- 发布记录 + 抓到的数据 → `05-复盘商业化/发布复盘.md`。

### 单次抓取步骤（受控浏览器）
1. 打开该账号主页 URL（见首页「账号」表）。
2. 拿主页笔记卡片——**每条笔记链接自带 token**（主页当场给，有效；不用存死链接）。
3. 找标题匹配那条，用它的**带 token 链接**直接打开（别点卡片触发弹窗，用链接打开更稳），等渲染。
4. 执行 JS 抓：赞/藏/评 = 笔记互动栏 3 个 count（排除评论区）；评论数从「共 N 条评论」。
5. 回 `05-复盘商业化/发布复盘.md`，填该行对应时间点（24h/48h/7天）格子，格式 `赞/藏/评`，如 `1059/267/164`。

## 流水线看板（_首页.md 的可视化层）

- **双击 `看板/启动看板.command`** → 自动开浏览器 → Kanban 5 列：选题池 → 写稿 → 审核 → 配音 → 发布。
- 卡片在哪列 = 这条选题走到哪步。点卡片按钮（「开始写稿 →」「送审核 →」「去配音 →」「去发布 →」「标记已发布 ✓」）**直接改回 `_首页.md` 的状态格**；点错有「← 退回」。也能在页面里直接**新增 / 编辑 / 删除**选题。改前自动备份到 `看板/.backups/`。
- **看板和首页是同一份数据**，不是第二个数据源：看板只是 `_首页.md` 的可视化读写层。
- 关掉服务（关 Terminal 或 Ctrl+C）就什么都不留。仅绑 127.0.0.1，不联网，零依赖（需 Node.js）。
